## About

Backend engineer focused on building systems that connect data infrastructure with AI capabilities. I design data pipelines, API surfaces, and retrieval systems that hold together under real operational constraints.

My current work centers on RAG architecture — bridging vector databases, embedding pipelines, and LLM interfaces into cohesive, queryable systems. I prioritize correctness, clear data flow, and observable behavior over abstraction complexity.

---

## Current Work

**RAG-Powered Chatbot System**

Building a production-ready conversational interface using Retrieval-Augmented Generation:

- Vector database architecture with Supabase (pgvector) for semantic search
- Complete CRUD pipeline for document ingestion and embedding management
- Context assembly layer: query → retrieval → prompt construction → LLM response
- Integration with OpenAI API for response generation with controlled context windows

Status: Core retrieval and generation pipeline operational. Refining context relevance scoring and response latency.
